# Brainberg Moderation Guide

This guide defines what belongs on Brainberg.eu. It is used by both human moderators and the AI moderation system.

## Platform Scope

Brainberg is a **European tech event aggregator**. We list events where people who build, use, or think about technology come together — in person or online.

**"Tech" is broad and domain-crossing.** Any event whose core focus is digital systems, computational methods, engineered hardware, or the technical craft and culture around them is in scope. That includes the topics our categories cover (software, game dev, AI/ML, data, cloud, security, policy/ethics, design & UX, blockchain, entrepreneurship, tech leadership & product, hardware/IoT, bio & health tech, hacker/maker) **and** cross-domain "*-tech" fields where technology or innovation is the lead: climate/sustainable tech, fintech, legaltech, edtech, agritech, govtech, spacetech, and so on.

Social, networking, and community events aimed specifically at this audience are also in scope (e.g. tech Stammtisch, founder drinks, OSS contributor dinner, repair cafés).

**No technical-content test for community-audience events.** If the organizer is a user group, hackerspace, tech community, founder collective, design collective, agile community, or any other group whose membership is recognizably "people who build, use, or think about technology", their drinks night, lean coffee, coworking day, beer meetup, or headshot social belongs on the platform. The test is **audience, not agenda**: would a European software engineer, founder, designer, or hacker plausibly attend? If yes, approve — even if the description is "come have a beer and talk shop" with no technical program. Require technical content only when the audience is genuinely generic (a random city business mixer using "tech" as a buzzword).

**"European" means:** events physically located in a European country (including UK, Iceland, Cyprus, Malta, and the European atlantic island groups), OR online events Europeans can actually participate in.

**Online = European by default.** An event on Zoom, Meet, Discord, YouTube, or any open platform that anyone can join is in scope, regardless of where the organizer or primary audience is based. Connecting European tech people to communities elsewhere (Africa, North America, Middle East, Asia) is a feature of the platform, not a problem. Non-European organizer, non-European audience focus, and non-European timezone are **not, by themselves, grounds for pending or reject**.

Only disqualify an online event when Europeans are explicitly excluded, e.g.:

- Registration is gated to a specific non-European geography ("US citizens only", "open to ACM NYC members")
- Benefits require local presence elsewhere (in-person hiring for a specific US city dressed up as an online event)
- Scheduled at a time no European could reasonably attend (3am–6am CET) *and* the description is explicitly about a local audience elsewhere

## Categories

Each event gets exactly one primary category. Pick the best fit based on the event's main focus.

### AI/ML Research & Engineering (`ai_ml_research`)

Building, training, and deploying machine learning models. MLOps, research papers, frameworks like PyTorch and Hugging Face, NLP, computer vision, reinforcement learning.

**Examples:** deep learning meetup, MLOps conference, PyTorch workshop, NLP paper reading group

### AI Integration & Application (`ai_applied`)

Applying AI rather than researching it. Covers both developers using AI tools (Copilot, Cursor, agents, vibe coding, LLM APIs, prompt engineering, no-code AI) and organizations integrating AI into products, workflows, or operations (AI for marketing, legal, HR, customer support, etc.).

**Examples:** "Build with AI" workshop, Claude Code meetup, AI agents hackathon, generative AI in business, AI for marketers, Copilot for lawyers

### Software Engineering (`software_dev`)

Programming languages, frameworks, architecture, testing, and the craft of building software — events where the core topic is how software is written, or the core activity is doing so.

**Examples:** RustConf, React meetup, Java user group, software architecture summit, code retreat

### Game Development (`game_dev`)

Game engines, game design, indie game development, playtesting, game art, game audio, game studios. Events where the primary output is a game — video, tabletop, or XR.

**Examples:** Godot Stammtisch, Unity user group, GameDev Stockholm, Indie Game Devs meetup, Playtest Party, Games Summit

### Data & Analytics (`data_analytics`)

Data engineering, business intelligence, databases, data pipelines, analytics platforms. Events focused on managing and analyzing data.

**Examples:** PyData meetup, Databricks conference, Power BI user group, dbt meetup, PostgreSQL day

### Cloud & DevOps (`cloud_devops`)

Cloud infrastructure, Kubernetes, CI/CD, platform engineering, SRE, observability. Events about running and scaling systems.

**Examples:** KubeCon, AWS user group, DevOpsDays, Terraform workshop, SRE meetup

### Security (`security`)

Cybersecurity, privacy, application security, penetration testing, threat modeling, compliance.

**Examples:** OWASP meetup, BSides conference, CTF competition, security architecture workshop

### Policy, Ethics & Governance (`policy_ethics`)

Tech policy, AI governance, AI safety, digital rights, algorithmic accountability, responsible AI, tech-and-law, and compliance as a topic (AI Act, NIS2, DORA, DSA). The audience is people thinking about how technology should be built, deployed, and regulated. Compliance **tooling** (security certifications, IAM products) stays in `security` or `cloud_devops`.

**Examples:** AI Governance: From Policies to Practice; Bristol AI Ethics; Lund/Malmö AI Safety; Designing Responsible AI; Realismo artificiale (AI & society panel)

### Design & UX (`design_ux`)

User experience, accessibility, design systems, usability testing, UI design. Events where the primary focus is on designing for humans.

**Examples:** UX conference, accessibility workshop, design system meetup, SmashingConf

### Web3 & Blockchain (`blockchain_web3`)

Blockchain technology, DeFi, smart contracts, decentralized systems, crypto. Events focused on decentralized technology.

**Examples:** Ethereum meetup, blockchain developer conference, DeFi workshop

### Entrepreneurship (`entrepreneurship`)

Events for **tech founders, tech investors, and early-stage startup operators**: fundraising, VC, growth, product-market fit, starting and scaling tech companies.

**Examples:** startup pitch night, founder drinks, angel-investor meetup, YC-style demo night, VC office hours

### Tech Leadership & Product (`leadership_product`)

Engineering leadership, tech-lead / staff-engineer / platform-lead content, CTO/CIO-level strategy, product management, agile/scrum, team practice and people management in tech organizations. For people who lead, structure, or productize tech work — the layer above hands-on building.

**Examples:** CTO Craft Con, Engineering Leaders meetup, CIO conference, ProductTank, Scrum/Agile meetup, itag's PM Forum, Clarity for Product Leaders, Manchester DevOps team-communication talks

### Hardware, Robotics & IoT (`hardware_iot`)

Physical computing, robotics, embedded systems, IoT, 3D printing, electronics, Arduino, drones, batteries. Events about building physical things.

**Examples:** robotics meetup, Arduino workshop, IoT conference, 3D printing demo, electronics lab

### Bio & Health Tech (`bio_health`)

Biotech, healthtech, medtech, digital health, bioinformatics, medical devices, life sciences — where the lead is the technology rather than the underlying science. Networking for practitioners, founders, and investors in these verticals belongs here too.

**Examples:** BioTech Connect Leipzig, HealthTech / MedTech Startups Networking, AI in Digital Health, Scaling Digital HealthTech, VDSG meets WiAI: AI-Ready Data in Healthcare

### Hacker / Maker Community (`hacker_maker`)

Hacker camps, hackerspaces, maker faires, community congresses (CCC, EMF, eth0). Events rooted in the hacker/maker culture and ethos. Do not apply a technical-depth test here: the hacker/maker culture spans absolute beginners to veteran hardware hackers. An event at a recognized hackerspace, fablab, or community innovation space (Waag, c-base, any member of hackerspaces.org) is in scope by default even when the description is thin.

**Examples:** Chaos Communication Congress, EMF camp, hackerspace open night, repair cafe, maker faire

### Other (`other`)

Tech events that are in scope for Brainberg but don't clearly fit any specific category above. Use this when no category applies — never force a fit into `software_dev` as a default. Human moderators will sort ambiguous-category events from here.

## Event Types

- **Conference** — multi-talk event, usually full-day or multi-day, with a program and speakers
- **Meetup** — informal community gathering, typically evening, 1-3 talks or discussions
- **Workshop** — hands-on, interactive learning session (bootcamp, masterclass, dojo)
- **Training** — paid, structured course (often multi-session or multi-day) leading to a certification or deep skill acquisition, run by a professional training provider rather than a community. Distinct from `workshop` (community-driven, one-off, typically free or cheap). Examples: ISTQB courses, Odoo Academy classes, AWS Certified Trainer courses, Quantum Training
- **Hackathon** — collaborative building event, usually timed (hack day, hack night, code retreat)
- **Networking** — social event primarily for meeting people (drinks, stammtisch, breakfast, mixer)
- **Coworking** — come-work-on-your-own-stuff session at a hackerspace or tech coworking space. Not `networking` (no explicit social pretext), not `hackathon` (no shared goal or deadline), not `meetup` (no program). Examples: Open Coworking Day, Programmiercafé, Code Café, Lunch Break Make, Cowork Cafe
- **Webinar** — online-only presentation or panel
- **Panel** — moderated discussion with multiple speakers
- **Demo Day** — showcase of projects or startups
- **Career** — hiring events (career fairs, recruitment days), plus career-development content for working tech professionals (resume reviews, interview workshops, senior-role transition talks, mentoring programs open to practitioners), and tech-worker career panels

## Event Moderation Decisions

### Approve

Clearly a tech event that fits the platform. The vast majority of events from our scrapers should be approved.

### Pending (needs human review)

Something is ambiguous. Use pending when:

- The title is in a language you're unsure about and the description is missing
- It could be tech-adjacent but might not have technical content (e.g., "Tech Networking Drinks" — is it for tech people or just a bar event using "tech" as a buzzword?)
- The event seems auto-generated or spammy but you're not sure
- It's a recurring event with a very generic title and no description

**When in doubt, use pending rather than reject.** A human can quickly approve a pending event, but a rejected event might be lost.

**Concrete test (pending vs approve):** if you can plausibly imagine a European tech founder, engineer, designer, or investor getting value from this event — and it's either physically in Europe or online — approve it.

Use **pending** only when there is a specific, concrete ambiguity you cannot resolve from the description or other info (e.g. "the title is in a language I can't read and there's no description", "the event description is two words", "format could be a cold sales pitch and there's no organizer info").

Defaulting to pending on weak signals clogs the human review queue and delays listings. Defaulting to approve is cheap to recover from — a human moderator can flip an approved event later.

### Reject

Clearly not a fit. Reject when:

- **Not tech:** the event has nothing to do with technology or creative/innovative building (e.g., yoga class, cooking workshop, live music, movie night, fashion show, poker night).
  - **Note:** of course there are exceptions in the form of high-tech, high-skill, or high-nerdiness variants of such event categories. E.g. we love an interactive jam session with DIY instruments (e.g. Look Mum No Computer), and of course the yearly screening of the movie Hackers (1995).
- **Spam or marketing:** the event is a thinly disguised sales pitch with no educational or community value
- **Student/graduate-only recruitment:** reject when the audience is explicitly fresh graduates, students, or early-career job-seekers in university contexts, and the core purpose is employer-student matching (campus career fairs, graduate hiring days, STEM-to-industry matching). Career development events for working tech professionals (interview prep, senior-role transitions, FAANG interview workshops, resume reviews, career pivots) are **in scope** under the `career` event type and should approve.
- **Non-European in-person:** the event is physically located outside Europe (online events from anywhere are fine)
- **Duplicate listing:** exact same event already on the platform under a different name

## Edge Cases

- **"Tech" in the name but not tech:** "Happy Techno" (DJ night), "Bake Tech" (baking), "Hair Tech" (salon) — reject
- **Tech-adjacent business events:** "FinTech Networking" — approve if organized by or for a tech/fintech community (per audience-not-agenda). Pending only if the organizer looks like a generic business-networking group using "fintech" as a buzzword.
- **Kids' coding events (CoderDojo):** approve — they're educational tech events
- **Open source community events (FOSDEM, Linux user groups):** approve — core tech community
- **Women/diversity in tech events:** approve — community building within tech
- **Barcamp / unconference:** approve — community-driven tech format
- **Digital literacy for seniors ("Smart Tech for Seniors"):** approve — tech education
- **Building/construction tech:** approve if there's a clear tech or engineering-innovation angle (smart buildings, modular construction, 3D-printed concrete, construction-software demos); reject if it's a general trade fair with no technical content
- **Sustainable/climate/green tech:** approve if the focus is on the technology (energy storage, grid software, carbon accounting platforms, climate-modelling ML, clean-energy hardware) — use the relevant tech category. Climate/tech policy events go under `policy_ethics`. Reject only pure political advocacy with no tech angle at all.
- **Domain + tech crossovers** (legal, ed, agri, gov, space, fintech, etc.): approve when the event is organized by or for practitioners/founders/investors in that tech vertical. Hands-on technical content is not required (per audience-not-agenda). Bio and health tech crossovers have their own category (`bio_health`).
- **Science conferences with no tech angle:** reject unless there's a clear technology/computing component
